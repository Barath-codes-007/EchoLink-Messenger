# 🔗 EchoLink Messenger

**A temporary, room-code chat app. No login. No signup. No database. When both people leave, the room — and every message in it — is gone for good.**

![EchoLink banner](assets/logo.svg)

---

## ✨ Features

### Home Page
- Modern glassmorphism UI with an animated gradient-orb background
- Username + room code entry
- One-click room code generator
- Copy-to-clipboard for the room code
- QR code generator for instant mobile join
- Dark mode with animated toggle (remembered via `localStorage`)
- Fully responsive layout

### Chat Page
- WhatsApp-style message bubbles with slide-in animation
- Real-time typing indicator
- Live online/offline status (waiting vs. both connected)
- Room code display + one-click copy
- Emoji picker (`emoji-picker-element`)
- Audio messages — upload button **or** drag-and-drop a file straight into the chat
- Voice messages — record directly from your microphone with a live timer, then send or discard
- Any file type — photos render as inline previews (click to view full-size), other files (PDFs, docs, zips, etc.) appear as a downloadable file card
- Clickable links — URLs typed in chat are automatically turned into tappable links
- Timestamps on every message
- Auto-scroll to the latest message
- System messages for join/leave events
- Leave-room button with confirmation

### Backend
- Node.js + Express + Socket.IO
- 100% in-memory — nothing touches disk or a database
- Rooms are capped at **8 people** by default (easy to change in `server/server.js` via `MAX_ROOM_SIZE`)
- A room and its entire message history are deleted the instant it becomes empty
- Server-side input validation and HTML-escaping on every message (XSS protection)

## 🚀 Installation

**Requirements:** Node.js ≥ 16

```bash
# 1. Clone or download the project
git clone https://github.com/your-username/EchoLink-Messenger.git
cd EchoLink-Messenger

# 2. Install backend dependencies
cd server
npm install

# 3. Start the server
npm start
```

The server serves both the API/Socket.IO layer **and** the static frontend files, so once it's running, open:

```
http://localhost:3000
```

For development with auto-restart on file changes:

```bash
npm run dev
```

## 🧭 Usage

1. Open the app in your browser.
2. Enter a display name.
3. Either generate a room code or type one you already have.
4. Share the code (or the QR code) with one other person.
5. Once they join, start chatting — text or drag in an audio clip.
6. Leave the room whenever you're done. As soon as both people are gone, the room and its messages are permanently erased from server memory.

---

## 🔒 Security Notes

- All incoming text messages are HTML-escaped server-side before broadcast (XSS mitigation).
- Room codes and usernames are validated on both client and server.
- Rooms are hard-capped at 8 participants by default (configurable via `MAX_ROOM_SIZE` in `server/server.js`).
- Audio uploads are restricted to `audio/*` MIME types and an 8MB size ceiling.
- No message, username, or room code is ever written to disk or a database — everything lives in a JavaScript `Map` in server memory and is discarded on room close.

## 📦 Deployment

EchoLink Messenger is a single Node process serving both static assets and WebSocket traffic, so it deploys cleanly to most Node-friendly hosts:

- **Render / Railway / Fly.io** — point the service at `server/server.js`, set `PORT` via the platform's env var (the app already reads `process.env.PORT`).
- **A VPS** — run behind Nginx as a reverse proxy with WebSocket upgrade headers enabled, and keep the process alive with `pm2` or `systemd`.
- **Docker** — a minimal `Dockerfile` would `COPY` the repo, `WORKDIR /app/server`, `RUN npm install --production`, and `CMD ["node", "server.js"]`.

Because there's no database, there's no migration step, no connection string, and no persistent volume required — deployment is just "install deps, run the process."
